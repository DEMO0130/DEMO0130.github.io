iPv6 那点事

iPv4 和 iPv6

为什么有iPv6？

原因很显然，因为iPv4不够用啦，当初设计iPv4是32位，显然是不够全世界那么多人、机构配一个IP的，于是iPv6闪亮登场

iPv4

iPv4是32位，十六进制表示法：XX:XX:XX:XX

iPv6

iPv6是128位，十六进制表示法：XXXX:XXXX:XXXX:XXXX:XXXX:XXXX:XXXX:XXXX，当有一段连续的0，可以用:压缩表示，比如 FF01:0:0:0:0:0:0:1101 → FF01::1101

iPv4 iPv6 映射规则

64:ff9b::IPv4 = IPv6

DNS64

在iPv6设备环境下，当传统DNS服务器把域名转换成iPv4地址传回来，这时候就需要把IPv4地址通过映射规则转换成一个看着像IPv6地址的地址

功能

把iPv4地址映射成iPv6地址

运作方式

  1. 判断IP，如果是iPv4，那么执行IP地址转换；如果是iPv6，不用转换
  2. 根据映射规则，映射即可

NAT64

在iPv6设备环境下，不管原IP地址是iPv6还是iPv4，如果不做转换，访问地址就真的变成iPv6格式的地址了，那些是iPv4的路由器显然就不认识你访问的是啥了，所以当真的要去访问外网资源时，还需要把IP地址转换成原IP地址

功能

把IP映射成原IP地址，然后使用原IP地址调用API

运作方式

  1. 通过映射规则，我们知道iPv4转换成iPv6后，前缀就是64:ff9b::
  2. 通过判断前缀，如果前缀等于64:ff9b::，那么执行IP地址转换；如果不等于，说明原IP就是iPv6格式的，不用转换
  3. 根据映射规则，反向映射即可
  4. 使用映射判断后的IP地址访问外网资源
