批量发布pod库

发表于 2018-03-03   |  

0x00 背景

每当一个开发周期完毕，app 发掉版本后，此次修改的 pod 库也需要发布，这时候人力一个一个去发，耗时耗力

于是弄个脚本来做这种事

0x01 前期准备

fastlane
这里使用 fastlane 来提供 pod 发布的一些具体 action 功能，如 pod lint、version increase、pod push
所以每个 pod 库里都需要进入 fastfile 文件来响应这些功能
而包含这些执行发布 action 的文件以引入形式引入到 pod 库里的 fastfile 文件

0x02 最终方案

0x0201 shell 脚本

脚本思路是这样

  1. 从主工程 podfile 中筛选出需要发布的 pod，存放到一个数组中
  2. 在主工程文件下生成一个 podRepos 文件夹用来存放 pod repo，循环3-8步
  3. 获取数组中的 pod 信息，将 pod clone 下来，进入对应的 pod 目录
  4. 检出响应的 feature 分支，执行 pod lint ，通过则继续，不通过则回到第3步继续下一个 pod
  5. 合到 develop，合到 master
  6. increse version
  7. pod push
  8. develop 合 master
  9. 回到 podRepos 目录
  10. 所有发完后删除 podRepos 文件

这里脚本还差最后一步没有实现，
所有 pod 发完后，主工程 podfile 文件应该将对应 pod 的 feature 指定给删掉
